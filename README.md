# Blackjack

I have come to believe that one of the best ways to kinda learn a language is to basically create a similar program in each one, and I believe that one of the best programs to do this is coding up Blackjack. It's a complex game, but still simple enough to be easily coded (I doubt any of these implementations, unless it's an online implementation with a GUI will reach over a thousand lines). The complexity means that a wide vareity of skills are required, albeit at a simple level: control flow statements, input handling (this may also require exception handling, after all you don't want someone to bet "Hello World!" dollars on Blackjack) as well as functions (and classes if desired).

The project is also pretty modular, since you can avoid some of the trickier aspects (splitting, insurance, a GUI, any online capabilities) for later and still have a functioning terminal game. 

This repository should include a version of blackjack in every programming language I am interested int, possible in multiple versions (an OOP or functional version, for example).
